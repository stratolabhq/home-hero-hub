Schema mismatch discovered:
  The scraper was designed with these fields: model_number, description, matter (bool)
  The actual Supabase products table has:     notes (merged field), no matter column,
                                              no model_number column, no description column

  Impact: The scraper's toRow() function merged model_number + description into notes,
  which worked correctly. However, there is no dedicated matter boolean column,
  so filtering "WHERE matter = true" is NOT possible in Supabase.
  Currently identified via product_id LIKE 'matter-%' pattern.

Image quality:
  The CSA-IoT WordPress REST API returned featured_media data, but 10,609 of
  10,609 Matter products have empty image_url. The API's _embed=wp:featuredmedia
  parameter returned null/empty source_url for virtually all products — the CSA
  registry does not store product images in WordPress featured media.

HTML entities in names:
  Many product names contain raw HTML entities that were not decoded:
    "WiZ LED luminaire &#8211; Full color"  → should be "WiZ LED luminaire – Full color"
    "WiZ LED strip &#8211; Full color"      → should be "WiZ LED strip – Full color"

5. ADDITIONAL ISSUES FOUND
================================================================================

5a. Non-smart-home devices in Matter import
    The CSA-IoT "Matter" category includes platform/SDK certifications:
    - Apple iOS, tvOS, watchOS, iPadOS, HomePod software, Darwin (~17 entries)
    - "Apple's Platforms" (3 entries)
    - These have type "Matter Device" and are not purchasable products.
    Recommended action: Filter out entries where type = 'Matter Device' AND
    brand = 'Apple Inc.' OR where name contains 'OS' / 'software' / 'Platform'.

5b. Reference design entries (chip vendor testing modules)
    - Quectel "On Off Light" ×91 — reference firmware on a chip module
    - Quectel "Extended Color Light" ×73 — same issue
    - Qorvo "QPG6105 DK" entries — development kit certifications
    These are legitimate CSA entries but represent chip vendor test/reference
    hardware, not consumer products users can buy.

5c. TP-Link Kasa cross-scraper overlap
    Kasa products exist twice in the DB:
      1. product_id "kasa-smart-plugs-kp125m" (from kasa-scraper, our data)
      2. product_id "matter-kasa-smart-plug-slim-energy-monitoring-kp125m" (from Matter import)
    The Kasa scraper entries are richer (have type, category, features).
    The Matter entries are redundant. Affected: ~10 Kasa products.

5d. Wyze cross-scraper overlap
    "Wyze Cam v3" appears twice: once from wyze-scraper, once from Matter import.

5e. HTML entity encoding not decoded in scraper
    The matter-scraper.ts stripHtml() function removes tags but does NOT decode
    HTML entities. Affects names and notes across ~200+ products.


================================================================================
6. RECOMMENDATIONS
================================================================================

PRIORITY 1 — Fix HTML entity decoding in matter-scraper.ts
  Add entity decoding to stripHtml(): replace &amp; → &, &#8211; → –, etc.
  This affects product names displayed to users.

PRIORITY 2 — Filter non-consumer entries before import
  In matter-scraper.ts, before saving, filter out:
    - Products where brand is a chip vendor (Quectel, Espressif, Qorvo, Beken, etc.)
      and name is a generic type ("On Off Light", "Extended Color Light", etc.)
    - Apple platform/OS entries (iOS, tvOS, watchOS, Darwin, etc.)
    - Any product with type "Software Component" (from CSA product_type taxonomy)

PRIORITY 3 — Deduplicate cross-scraper entries
  After all scrapers run, run a deduplication pass:
    - If a product exists as "kasa-xxx" AND "matter-xxx", keep "kasa-xxx" (richer data)
    - If a product exists as "wyze-xxx" AND "matter-xxx", keep "wyze-xxx"
    - Similar for Philips Hue (philips-hue-xxx vs matter-xxx)

PRIORITY 4 — Normalize category casing
  Aqara and other manually-imported products use lowercase category slugs
  ("sensors", "hubs") while scrapers use Title Case ("Sensors", "Hubs & Controllers").
  Standardize to Title Case across all records.

PRIORITY 5 — Add matter boolean column to schema
  The products table lacks a `matter` boolean column. Currently Matter products
  are identified by product_id prefix 'matter-%', which is fragile.
  Consider: ALTER TABLE products ADD COLUMN matter boolean DEFAULT false;
  Then update existing: UPDATE products SET matter = true WHERE product_id LIKE 'matter-%';
  Also update Kasa and other scrapers to set matter = true on Matter-capable devices.

PRIORITY 6 — Images for Matter products
  The CSA-IoT REST API has no product images. Options:
    - Accept no images for Matter entries (cert database, not product catalog)
    - Attempt image lookup by brand+name via a secondary source
    - Show a generic Matter logo as fallback image
